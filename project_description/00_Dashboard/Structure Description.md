### Структура папок (Folder Structure)

*   **`00_Dashboard`** — Головна нотатка (MOC - Map of Content), план робіт, Kanban-дошка.
*   **`01_Vision_&_Concept`** — Візія, аналіз ринку, ключові когнітивні принципи.
*   **`02_Requirements`** — User Stories, функціональні вимоги, нефункціональні вимоги.
*   **`03_Architecture`** — Технологічний стек, схеми БД, логіка AI-агентів, API-специфікації.
*   **`04_Design`** — Описи інтерфейсів, посилання на Figma, логіка UX-шляху.
*   **`05_Research_&_Science`** — Твої нотатки по книзі *Make It Stick*, посилання на статті про RAG, ШІ в освіті (тут і будуть твої Zettelkasten-нотатки). Також нотатки для диплому
*   **`06_Development_Log`** — Щоденні звіти (Daily Notes) про те, що зроблено, які помилки виникли.
*   **`07_Assets`** — Схеми (PNG/SVG), PDF-джерела, скріншоти.

    00_Dashboard: Тут лежить головна нотатка Index.md (план на день) та твоя Kanban-дошка.

    01_Theory: Тут ти збираєш базу для диплома. Конспекти книги «Make It Stick», статті про ШІ, RAG, когнітивну психологію.

    02_Requirements: Твої User Stories та Функціональна декомпозиція (список усіх фіч).

    03_Architecture: Схеми (Mermaid), опис бази даних, API, промпти для ШІ.

    04_Design: Опис інтерфейсу, скріншоти з Figma або начерки з Excalidraw.

    05_Academic_Thesis: Твій текст диплома. Розділ 1, Розділ 2 і т.д.

    06_Dev_Log: Щоденник розробки. Щодня пиши 2 речення: «Що зробив» і «Яку проблему зустрів». Це база для розділу «Реалізація».
