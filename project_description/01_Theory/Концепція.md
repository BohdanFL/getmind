# Концепція платформи «CogniFlow»
**AI-driven екосистема для глибокого дослідження та довготривалого засвоєння знань**

## 1. Візія продукту
**CogniFlow** — це не просто платформа для читання курсів, а «когнітивний тренажер», побудований на принципах доказової когнітивістики (*Make It Stick*). Платформа трансформує пасивне споживання інформації в активне дослідження, змушуючи мозок учня долати «бажані труднощі» для вибудовування нейронних зв'язків, які зберігаються роками.

## 2. Ключова проблема (The Gap)
Більшість LMS та ШІ-тьюторів діють як «інтелектуальні милиці», що полегшують навчання (роблять резюме за учня). Це створює **ілюзію компетентності**. Натомість CogniFlow використовує ШІ, щоб змусити користувача самостійно генерувати рішення, пояснювати концепції та виявляти зв'язки.

---

## 3. Етапи користувацького шляху

### Етап I: Підготовка та Структурування (The Knowledge Map)
*   **Вхідні дані:** Користувач завантажує PDF-файли або текстові матеріали.
*   **Процес:** ШІ проводить семантичне розчленування тексту на логічні блоки (топіки) та виявляє зв'язки між ними.
*   **Результат:** Платформа генерує **Інтерактивну карту знань** — візуальний граф, де видно ієрархію тем. Користувач може підкоригувати структуру перед початком навчання.
*   **Додаткова фіча (Audit):** Оцінка якості матеріалу на повноту та пропозиція доповнити прогалини з бази знань ШІ.

### Етап II: Процес навчання (The Study Cockpit)
Інтерфейс розділений на три взаємопов'язані зони:

1.  **Smart PDF Viewer (Ліва панель):**
    *   Читання матеріалу з автоматичною зупинкою на **Cognitive Checkpoints**.
    *   Після кожного блоку/сторінки інтерфейс блокується для **Retrieval Practice** (коротке відкрите питання від ШІ, на яке треба відповісти, не заглядаючи в текст).

2.  **Socratic AI Tutor (Права панель):**
    *   Чат-дослідник, який має «Pedagogical System Prompt». Він не дає прямих відповідей, а скеровує користувача за допомогою навідних запитань, посилаючись на конкретні сторінки документа (**Source Anchoring**).

3.  **Elaboration Notes (Розумні нотатки):**
    *   Користувач пише свій конспект. ШІ аналізує його в реальному часі.
    *   Якщо нотатка — це копіпаст, ШІ просить перефразувати. Якщо бракує прикладів — просить навести власну метафору.

### Етап III: Практичне застосування (The Project Lab)
Після опанування 3–5 тем система ініціює «Проєктну лабораторію».
*   **Механіка:** Симуляція реального кейсу (наприклад, рольова гра «Маркетинговий директор» або «Системний архітектор»).
*   **Принцип:** Для вирішення кейсу користувач має одночасно використати знання з різних розділів матеріалу (**Interleaving**).
*   **Фідбек:** ШІ виступає в ролі замовника або опонента, який перевіряє не просто знання фактів, а здатність до синтезу.

### Етап IV: Утримання знань (The Retention Loop)
*   **Visual Analytics:** Граф знань «тьмяніє» з часом, візуалізуючи криву забування.
*   **Spaced Repetition:** Автоматична генерація запитань для повторення через інтервали (1 день, 3 дні, тиждень), що надсилаються користувачу.

## 4. Когнітивний фундамент (Методологія)

| Метод | Реалізація в CogniFlow |
| :--- | :--- |
| **Active Recall** | Примусові паузи на питання під час читання (Checkpoints). |
| **Spaced Repetition** | Алгоритм FSRS для повторення тем, що забуваються. |
| **Interleaving** | Змішування питань з різних тем у Project Lab. |
| **Elaboration** | Аналіз нотаток на унікальність та глибину пояснення. |
| **Generation** | Спроба розв’язати кейс у Project Lab до отримання правильної відповіді. |
| **Desirable Difficulties** | ШІ-тьютор, який «не каже відповідь», змушуючи мозок працювати. |

## 5. Технологічний стек (Рекомендація)
*   **LLM:** Gemini 1.5 Pro (завдяки контекстному вікну 2M токенів) або GPT-4o.
*   **Framework:** LangChain або LangGraph (для побудови циклічної логіки тьютора).
*   **RAG Engine:** Pinecone / ChromaDB для зберігання та пошуку по завантажених PDF.
*   **Frontend:** React / Next.js із PDF.js інтеграцією.
*   **SRS:** Інтеграція алгоритму FSRS-rs.

---

## 6. Висновок для дипломної роботи
Проєкт **CogniFlow** вирішує проблему низької ефективності сучасної освіти через автоматизацію технік когнітивної психології. Його наукова новизна полягає у переході від LLM як «відповідача» до LLM як «архітектора навчальних зусиль», що забезпечує довготривалу зміну когнітивних структур учня.
